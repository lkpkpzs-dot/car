🏢 企业端 - 道路许可申请

三类申请的完整流程

⚠️ 重要前置条件

  • 必须先完成企业资质申请并通过审核
  • 必须提前注册安全员并获得资质证书
  • 道路许可申请需要指定负责的安全员

🛣️ 道路许可申请类型

申请类型 适用场景 审批周期 所需安全员
道路测试 新车种、新技术上路测试 3-5个工作日 至少1名
示范应用 限定区域示范运营 3-5个工作日 至少1名
应用试点 规模化商业运营试点 5-7个工作日 至少2名

申请提交流程

1. 选择申请类型
根据业务需求，选择道路测试、示范应用或应用试点
2. 填写申请信息
企业基本信息、车辆信息、申请路段、时间周期、指定安全员等
3. 上传申请材料
营业执照、车辆合格证、保险凭证、安全承诺书等
4. 提交申请
确认无误后提交，等待车管所审核

🏛️ 车管所端 - 道路审核

审核、查验、发牌一站式服务
1
车管所
审核申请材料
审查企业资质、车辆信息、安全员资质、申请材料完整性
→
2
车管所
车辆查验
现场或线上查验车辆状况，确认符合要求
→
3
车管所
发放号牌
审核通过后，发放专用车辆号牌
→
4
车管所
完成审批
生成许可文件，通知企业领取

审核处理结果

✅ 审核通过

  • 车辆查验合格
  • 安全员资质有效
  • 材料齐全有效
  • 发放专用号牌
  • 企业可开展运营

⚠️ 审核不通过

  • 说明不通过原因
  • 告知需要补充的材料
  • 企业可重新申请
  • 记录审核历史

⏰ 超时升级机制

举报超时

  • 企业超过3天未处理 → 自动提醒
  • 超过7天未处理 → 升级为车管所督办

审核超时

  • 资质/许可超过5天未审核 → 发送待办提醒
  • 超过10天未审核 → 升级为警督关注

📝 字段说明与超时机制

核心业务字段详解与系统超时规则

📋 核心业务字段说明

举报编号（Report ID）
格式：JB + 年月日 + 序号（例：JB20260611001），全局唯一标识
申请编号（Application ID）
格式：SQ + 类型代码 + 年月日 + 序号（例：SQRD20260611001），类型代码：RD-道路，ZZ-资质，AQ-安全员
许可类型（Permit Type）
1-道路测试、2-示范应用、3-应用试点，不同类型审核标准和安全员要求不同
安全员资质（Safety Officer Qualification）
A类-可负责所有类型，B类-仅可负责示范应用和应用试点，C类-仅可负责应用试点

⏰ 系统超时与升级规则

业务类型 阶段 首次提醒 二次升级 超时处理
市民举报 企业处理 3天 7天（督办） 记录企业超时次数
资质申请 车管所审核 5天 10天（警督） 纳入审核员绩效
道路许可 车管所审核 5天 10天（警督） 纳入审核员绩效
安全员注册 车管所审核 3天 7天（警督） 纳入审核员绩效

⚠️ 企业信誉扣分规则

  • 举报超时未处理：每次扣2分
  • 审核发现违规成立：每次扣5分
  • 提供虚假材料：每次扣10分，暂停申请资格3个月
  • 信誉分低于60分：进入重点监管名单
